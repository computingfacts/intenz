COLUMN_NAME DATA_TYPE NULLABLE DATA_DEFAULT COLUMN_ID COMMENTS
ENZYME_ID NUMBER(7,0) No null 1 Enzyme_id is used as an internal unique identifier throughout the database as some entries don't have an EC number, e.g. suggested entries. Example is '70001', '9945', etc.
EC1 NUMBER(1,0) Yes null 2 The first number, ec1, is the number of the class, example '3'.
EC2 NUMBER(2,0) Yes null 3 The second number, ec2, defines the subclass inside the class, example '3.1'.
EC3 NUMBER(3,0) Yes null 4 The third number, ec3, defines the subsubclass inside the subclass, example '3.1.1'.
EC4 NUMBER(4,0) Yes null 5 The fourth number, ec4, is a serial number of the enzyme inside the subsubclass, example '3.1.1.1'.
HISTORY VARCHAR2(400 BYTE) Yes null 6 Short form of history of an enzyme, especially date of creation, e.g. 'EC 1.1.1.1 created 1961', 'EC 1.1.3.8 created 1965, modified 2001'.
STATUS VARCHAR2(2 BYTE) No null 7 Internal code of status, for example 'NO', 'UN', 'SN', 'OK', etc.
NOTE VARCHAR2(2000 BYTE) Yes null 8 Comment or note of an enzyme annotator which wont be viewable for the public.
SOURCE VARCHAR2(6 BYTE) No null 9 Database internal code, for example 'INTENZ', 'IUBMB', 'BRENDA', 'SIB' etc. Each enzyme belongs to one of them.
ACTIVE VARCHAR2(1 BYTE) No 'Y' 10 IUBMB uses 'deleted' for different meanings. We say 'active' for enzymes that are currently recommended by IUBMB. Opposit of inactive
